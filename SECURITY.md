# 安全与隐私指南

## 概述

本文档描述了 OfficeCLI 的安全特性、已知限制以及如何安全地使用本工具。

## 安全特性

### 1. 禁用自动更新

商用版本已**完全禁用自动更新功能**，以防止供应链攻击。

- 不会自动下载或执行任何外部代码
- 所有更新必须通过手动方式安装
- 移除了自动更新检查的网络请求

### 2. 本地操作优先

- 所有文档处理在本地完成
- 不会上传文档内容到任何远程服务器
- 网络请求仅限于用户明确发起的操作（如下载图片）

### 3. 配置文件安全

- 配置文件存储在用户主目录下的 `.officecli/` 目录
- 建议设置适当的文件权限：
  ```bash
  chmod 700 ~/.officecli
  chmod 600 ~/.officecli/config.json
  ```

### 4. 日志隐私

- 日志功能默认**禁用**
- 启用后，日志中不会记录完整的文件路径（仅记录文件名）
- 敏感内容（如密码、密钥）不会被记录

### 5. Watch服务器安全

- 仅绑定到本地回环地址 (127.0.0.1)
- 不会接受外部网络连接
- 建议在使用完毕后运行 `officecli unwatch <file>` 关闭

## 安全使用建议

### 1. 文件权限

确保 OfficeCLI 二进制文件和配置目录具有适当的权限：

```bash
# Linux/macOS
chmod 755 /path/to/officecli
chmod 700 ~/.officecli
```

### 2. 敏感文档处理

处理包含敏感信息的文档时：

- 避免在共享环境中使用 `watch` 功能
- 定期清理临时文件（位于系统临时目录）
- 使用 `validate` 命令检查文档是否包含意外元数据

### 3. 网络环境

- 在受信任的网络环境中使用
- 如果不需要网络功能，可以配置防火墙阻止 officecli 的网络访问

## 隐私注意事项

### 收集的数据

OfficeCLI **不会收集**以下数据：
- 文档内容
- 使用统计
- 遥测数据
- 崩溃报告

### 本地存储的数据

仅在本地存储以下信息：
- 用户配置偏好（位于 `~/.officecli/config.json`）
- 可选的命令历史日志（如果启用）

### 第三方服务

商用版本**不会**连接以下服务：
- 自动更新服务器
- 遥测/分析服务
- 云服务

## 已知限制

1. **MCP服务器**：通过 stdio 通信，依赖操作系统的进程隔离
2. **Watch服务器**：本地HTTP服务器，无身份验证（仅本地访问）
3. **命名管道**：常驻模式使用命名管道，依赖文件系统权限

## 报告安全问题

如果您发现安全问题，请：

1. 不要公开披露
2. 私下联系维护者
3. 提供详细的复现步骤
